# Development Guide

This document covers how to set up, run, and work on ManualMoney locally.

## Prerequisites

| Tool | Version | Notes |
|------|---------|-------|
| Java | 8+ | Required for the Spring Boot backend |
| Maven | 3.6+ | Included via Maven wrapper (`./mvnw`) |
| Node.js | 16+ | Required for the React frontend |
| npm | 7+ | Comes with Node.js |

## Initial Setup

### Backend

```bash
cd backend
./mvnw clean install
```

This downloads dependencies, compiles the code, and runs tests. The Maven wrapper (`./mvnw`) is checked into the repo so you don't need Maven installed globally.

### Frontend

```bash
cd frontend
npm install
```

## Running Locally

Start both the backend and frontend. The backend must be running for the frontend to work.

### Start the backend (port 8080)

```bash
cd backend
./mvnw spring-boot:run
```

### Start the frontend (port 5173)

```bash
cd frontend
npm run dev
```

Open `http://localhost:5173` in your browser. The Vite dev server proxies all `/api` requests to `http://localhost:8080` automatically.

## Configuration

### Backend

Configuration lives in `backend/src/main/resources/application.properties`:

```properties
server.port=8080
manualmoney.data.path=./data/manualmoney.json
```

- `server.port` — The port Spring Boot listens on.
- `manualmoney.data.path` — Path to the JSON data file. Created automatically on first write if it doesn't exist.

### Frontend

The Vite dev proxy is configured in `frontend/vite.config.ts`:

```typescript
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

The API client in `frontend/src/api/client.ts` uses `baseURL: '/api'`, so all requests go through the proxy during development.

### CORS

`backend/src/main/java/com/manualmoney/config/CorsConfig.java` allows requests from `http://localhost:5173` and `http://localhost:3000`. Update this if you change the frontend port.

## Building for Production

### Backend

```bash
cd backend
./mvnw clean package -DskipTests
java -jar target/manualmoney-1.0.0.jar
```

### Frontend

```bash
cd frontend
npm run build
npm run preview    # optional: preview the build locally
```

The production build outputs to `frontend/dist/`.

## Running Tests

### Backend

```bash
cd backend
./mvnw test                        # Run all tests
./mvnw test -Dtest=ClassName       # Run a single test class
./mvnw verify                      # Run tests + generate coverage report
```

Coverage report: `backend/target/site/jacoco/index.html`

### Frontend

```bash
cd frontend
npm test                           # Run all tests once
npm run test:watch                 # Run tests in watch mode
```

Coverage report: `frontend/coverage/index.html`

## Project Conventions

### Code Style

- **Backend**: Standard Java conventions. PascalCase for classes, camelCase for methods and fields. No linter or formatter is configured; follow existing patterns.
- **Frontend**: PascalCase for components, camelCase for functions and variables. TypeScript strict mode is enabled. No ESLint or Prettier is configured.

### Monetary Values

All money amounts use `BigDecimal` on the backend to avoid floating-point errors. The frontend receives these as numbers from JSON but does not perform arithmetic — all calculations happen server-side.

### Data File

The single JSON file at `backend/data/manualmoney.json` is the entire data store. It is loaded into memory on startup and rewritten after every mutation. Do not edit this file while the backend is running.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Frontend shows network errors | Make sure the backend is running on port 8080 |
| Port 8080 already in use | Stop the other process or change `server.port` in `application.properties` |
| Port 5173 already in use | Vite will auto-increment to the next available port |
| Data file not found on startup | The backend creates it automatically on the first write operation |
| Maven wrapper permission denied | Run `chmod +x ./mvnw` (Unix) or use `mvn` directly if Maven is installed globally |
